# Phase 2: Action Banner UI

## Goal

Add a compact chart-level banner that explains hidden drive/search states and provides direct actions for common recoverable causes.

## Scope

- Add banner markup near the existing chart diagnostic area in `tools/drive_comparison_template.html`.
- Add banner styles in `tools/drive_comparison_styles.css`.
- Add banner model, rendering, reason copy, and actions in `tools/drive_comparison_client/chart/interaction.js`.
- Use existing state and sync helpers from `tools/drive_comparison_client/state/core.js`, `tools/drive_comparison_client/presets/library.js`, and `tools/drive_comparison_client/calc/filtering.js`.
- Update `tools/drive_comparison_i18n.py` only if new static template text requires English replacement mappings.
- Regenerate published output through `npm run build`.

## Non-Goals

- Do not add hidden/ghost search points to the chart.
- Do not redesign the left filter panel.
- Do not introduce a new global state manager.
- Do not automatically lower target dV.

## Affected Files

- `tools/drive_comparison_template.html`
- `tools/drive_comparison_styles.css`
- `tools/drive_comparison_client/chart/interaction.js`
- Potentially `tools/drive_comparison_i18n.py`
- Generated by build: `docs/index.html`, `docs/assets/js/**`
- This phase document

## Implementation Steps

1. Add a dedicated `filterActionBanner` container near `chartDiagnostic`.
2. Add compact responsive styles for the banner, copy area, and action button row.
3. Add `filterActionBannerModel(diagnostics)` with priority:
   - active search with hidden total matches and no visible matches
   - active search with some hidden matches
   - no active search with hidden rows
   - otherwise hidden
4. Add user-facing reason labels for all action-relevant reasons:
   - `minTwr`
   - `minDv`
   - `targetDvOrMassRatio`
   - `familyFilter`
   - `thrusterCountFilter`
   - `researchFilter`
   - `axisRange`
   - `invalidPowerPlant`
   - `invalidComputation`
   - `other`
5. Add action definitions:
   - reset acceleration threshold
   - show impractical candidates
   - reset minimum dV
   - reset drive filters
   - reset engine count
   - clear search
6. Wire action buttons to mutate `state`, call `syncUiFromState()`, and let normal render flow update the banner.
7. Call `updateFilterActionBanner(diagnostics)` from the existing `render()` path.
8. Ensure English copy uses Acceleration terminology and Korean mode uses Korean copy.

## Acceptance Criteria

- Banner is hidden in normal default states where no useful action is needed.
- Banner appears when current settings hide meaningful results.
- Search-specific messages take priority over generic hidden-state messages.
- Banner identifies the dominant hidden reason.
- Reset acceleration threshold resets `state.minTwr` to `DEFAULT_MIN_TWR` and syncs controls.
- Show impractical candidates sets `state.showImpracticalCandidates = true` and syncs controls.
- Clear search empties `state.searchTerm` and syncs the search input.
- Existing `chartDiagnostic` behavior remains intact.

## Validation Commands


## Manual Smoke Tests

1. Open the chart.
2. Set metric to total mass.
3. Raise minimum acceleration until many drives disappear.
4. Confirm the banner appears and mentions acceleration.
5. Click reset acceleration threshold and confirm the threshold input/readout updates.
6. Raise minimum acceleration again and click show impractical candidates; confirm checkbox/state sync.
7. Search for a known drive, then hide it through filters or acceleration threshold; confirm search-specific copy.
8. Click clear search and confirm the search-specific banner disappears.
9. Switch English and Korean; confirm the banner changes language.

## Rollback Risks

- Incorrect action handlers could reset too much state. Keep each handler narrowly scoped.
- Calling `render()` directly from action handlers could double-render if `syncUiFromState()` already renders. Prefer `syncUiFromState()` as the single sync path.
- Static template strings may need `drive_comparison_i18n.py` mappings if they are visible before dynamic JS runs.
